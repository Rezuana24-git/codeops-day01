1. Git is a version control system that tracks changes to files on my computer. 
GitHub is a website that stores Git repositories online so I can back them up, share them, and collaborate with others.
2. The terminal allows me to interact with my computer by typing commands instead of using the mouse. It helps me create files, navigate folders, and use tools like Git.
3.`git add` stages changes. It tells Git which files or changes should be included in the next commit.
4.`git commit` saves a snapshot of the staged changes in the Git repository. Each commit records the project's history so changes can be reviewed or restored later.
 5.
1. Edit files.
2. Check changes using `git status`.
3. Stage changes using `git add`.
4. Commit changes using `git commit`.
5. Push changes to GitHub using `git push`.
